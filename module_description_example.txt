You need to generate a module description following this requirements:
- topmodule: the name of the top module
- Description: a short description of the module
- Submodules: the names of the submodules(Only include when make direct connections between submodules)
- Ports: the ports of the module, Define in the format of (input: <port_name>, ...) (output: <port_name>, ...)
- Connections: the connections between the ports, You need to be specific about how the connections are made. Include both internal and external connections.
- Parameters: the parameters of the module
Try to de compose the module into smaller modules if possible. When defining submodules you must also include the ports connections, descriptions. Note that the submodules may also exist on other modules, and if there isn't one you put submodules: Null. Otherwise you put the name and ports of the module there, splitting with ;
Also if such port can take multiple of bits, you must specify the width of the port also.

Example:
module: mux4to1
Description: select 1 from 4 input signals and output by calling on mux2to1
Submodules: mux2to1(input: a, b, sel) (output: y);
Ports: (input: i0, i1, i2, i3, sel1, sel2) (output: y)
Connections: Select one of i0, i1, with sel1 using mux2to1. Select one of i2, i3, with sel1 using mux2to1. Select one of the previous two outputs with sel2 using mux2to1 and output.
Parameters: Null

module: mux2to1
Description: select 1 from 2 input signals
Submodules: Null
Ports: (input: a, b, sel) (output: y)
Connections: If sel is 0, output a; if sel is 1, output b.
Parameters: Null
